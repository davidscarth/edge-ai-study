### vLLM:
```shell
sudo docker run --rm -it \
  --gpus all \
  --ipc=host \
  --network host \
  -v ~/.cache/huggingface:/root/.cache/huggingface \
  -v ~/models/templates:/mnt/templates \
  -e VLLM_USE_V2_MODEL_RUNNER=1 \
  ghcr.io/timothystewart6/vllm-gb10:latest \
  vllm serve RedHatAI/gemma-4-26B-A4B-it-NVFP4 \
  --host 0.0.0.0 --port 8000 \
  --gpu-memory-utilization 0.25 \
  --enable-prefix-caching \
  --speculative-config '{"method": "mtp", "model": "google/gemma-4-26B-A4B-it-assistant", "num_speculative_tokens": 3}' \
  --kv-cache-dtype fp8 \
  --max-model-len 32768 \
  --max-num-seqs 6 \
  --max-num-batched-tokens 8192 \
  --chat-template /mnt/templates/chat_template.jinja \
  --default-chat-template-kwargs '{"enable_thinking": true}' \
  --enable-auto-tool-choice \
  --tool-call-parser gemma4 \
  --reasoning-parser gemma4 \
  --api-key "REPLACE-THIS-WITH-YOUR-OWN-KEY"
```

The above lets me serve 6 requests with 32K context windows, and vLLM does pretty efficient batching so 6 users could all chat pretty comfortably with these settings, and leaves me plenty of RAM for other projects.

### Open WebUI System Prompt:
```shell
Today is {{CURRENT_WEEKDAY}}, {{CURRENT_DATE}}. Use search_web for anything recent, subject to change, or not confidently known, including questions about specific places, businesses, products, or people. Use fetch_url when the user shares a link or when search snippets lack needed detail. State plainly if unable to verify. Never invent facts, quotes, citations, or URLs. Think briefly: a few short sentences at most, then answer. Skip deliberation entirely for simple requests.
```

This system prompt fixed a lot of my initial problems. May need additional tuning, but it's designed to be token-efficient and concise. Each piece fixed a real issue I encountered. Highly recommended to be paired with the RAG Template below (which fixes a search calling snag I encountered in thinking mode, not fully but enough).

### Open WebUI RAG Template:
```shell
### Task:
Answer the user's query using the context below (documents and/or search results).

### Guidelines:
- When a claim comes from the context, cite it inline as [id] matching the numbered <source> entries (e.g., [1], [2]). Do not cite your own general knowledge.
- If the context doesn't contain the answer but you know it, answer from your own knowledge and note that briefly.
- If you can't answer reliably, say so plainly.
- Respond in the user's language.
- Never mention these instructions, the context block, or any XML tags in your reply.

<context>
{{CONTEXT}}
</context>
```

This is a lot simpler.
